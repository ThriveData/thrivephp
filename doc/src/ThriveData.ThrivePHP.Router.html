    Class ThriveData\ThrivePHP\Router

    Routes HTTP requests from a front controller to registered Route objects. This is a static singleton class.

    This class functions based on the Front Controller pattern, where a webserver (such as nginx) is configured to proxy HTTP requests to a front contoller.

    The front controller passes the request from the webserver to Router::call().

    Router::call() searches the registered Route objects (via Router::register()) for a Route that matches the HTTP method, host, and URL.

    When a match is found, the callback given in the Route object is called, which is typically a method in a subclassed Page or API class.

    If a match is not found, a HTTP 404 response is returned.
